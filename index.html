Coding Challenge:

Distributed Task Manager

Let's imagine that you are working for a company that deals with big data. The company runs heavy computational tasks that can be executed in parallel and independently. You've been tasked with designing and implementing a distributed task manager in JavaScript to control and manage the execution of these tasks in an efficient and structured way.

Here are the specific requirements:

  • The Task Manager should be implemented as a class named DistributedTaskManager.
  • The class should have a method named addTask which accepts an asynchronous function (representing a task) and a unique identifier for this function. If a duplicate identifier is given, it should throw an error.
  • The class should have a method named executeTasks that starts the execution of all added tasks in the order they were added.
  • The executeTasks method should return a promise that resolves with an object that maps identifiers to their resolved values. If a task promise is rejected, it should be included in the output object as well but marked clearly as an error.
  • The class should include a method named executeTasksWithLimit which behaves similarly to executeTasks, but accepts a parameter n that specifies the maximum number of tasks that can be running simultaneously.
  • Write unit tests for this class that cover normal operation as well as edge cases and potential exceptions.
